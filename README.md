# Domain-Driven-Design in Java

This is the repository for the workshop Domain-Driven-Design in Java

## Development

run `./gradlew bootRun`

If you want a continuous build on code change, you can run these two commands in separate terminals:
- `./gradlew build --continuous`
- `./gradlew bootRun`

## Testing

run `./gradlew test`

## Internals

Everything that's named internal is to be considered private.
If you ever have to touch something in an internal folder, you should consider this a bug and should report it.
